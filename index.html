Flujos de trabajo

Usa tabs para comparar estrategias de trabajo en equipo.

Una rama principal, ramas cortas por tarea y merges frecuentes.

  • Ideal para equipos pequenos
  • Requiere disciplina en naming
  • Permite releases rapidos

Separacion estricta entre desarrollo, releases y hotfix.

  • Bueno para productos con ciclos largos
  • Releases planificadas
  • Mayor control sobre versiones

Integracion continua en una rama principal con feature flags.

  • Entrega continua
  • Commits pequenos y frecuentes
  • Requiere pruebas automaticas

Cheat sheet interactivo

Filtra y copia comandos para tus practicas.

0 comandos
git status

Revisa el estado del area de trabajo.

git add -p

Agrega cambios por partes para commits limpios.

git commit -m "mensaje"

Crea un commit con mensaje corto.

git log --oneline

Resumen rapido del historial.

git checkout -b feature/nueva

Crea una nueva rama y cambia a ella.

git merge feature/nueva

Fusiona cambios de otra rama.

git rebase -i HEAD~3

Reordena, edita o combina los ultimos commits.

git stash push -m "avance"

Guarda trabajo temporalmente sin hacer commit.

git tag -a v1.0 -m "release"

Crea una version etiquetada.

git reset --soft HEAD~1

Vuelve un commit atras manteniendo staging.

git revert <hash>

Revierte cambios creando un nuevo commit.

git push origin main

Sube tus cambios al remoto.
